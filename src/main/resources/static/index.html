Traffic congestion detector

It is the year 2040 the number of cars on the road has reached almost unsustainable levels.
The government establishes a new Traffic Officer department, and you are members of the advanced R&D division.

Our division is researching the use of drone beacons with Automated Number Plate Recognition (ANPR).
The drones can rest on platforms along roads and motorways and can be relocated in a matter of minutes.

Your tasks

Coding challenge

Develop an API that calculates the average car speed in a particular area.
If the average speed falls below 10mph, a congestion alert is raised.

Some cars will not get past the second ANPR collector, ensure that does not impact the average speed calculation or introduce memory leaks.

Operations/Networking challenge

Develop a Dockerimage for your application. Build, deploy to docker hub so that it runs on the pi cluster.

Stretch goal

If the mobile beacon is re-located, the distance between the ANPR systems changes, recalculate the distance and calculate the average speed appropriately from then on.

Hints

Speed is calculated as distance/time. Distance could be miles/km/m;

Your software will be deployed in a cluster, decide early your state strategy (Stateless, stateful services);

(If applicable) Decide which part of the software delivery life cycle is going to be responsible for code compilation;

Raspberry pis have a specific architecture. It might influence compilation strategy.
